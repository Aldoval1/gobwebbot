## 2024-03-15 - Map Controls Accessibility
**Learning:** Custom interactive map elements in templates (like `.zoom-btn` in `licenses.html` and `official_businesses.html`) are frequently implemented as non-semantic `<div>` elements with `onclick` handlers, rendering them completely inaccessible to keyboard and screen reader users. Simply adding `:hover` states isn't enough; map controls require explicit focus states, semantic HTML, and descriptive ARIA labels to be truly accessible.
**Action:** When refactoring custom interactive UI elements, always replace non-semantic tags like `<div>` with `<button type="button">`. Ensure they feature descriptive `aria-label` attributes localized to Spanish, add `aria-hidden="true"` on inner decorative icons (like FontAwesome), and add explicit `:focus-visible` CSS rules to maintain consistent keyboard accessibility across the application.
