## 2024-03-24 - Map Zoom Control Accessibility Refactoring
**Learning:** Found custom map zoom controls implemented as `div` elements with `onclick` handlers lacking keyboard accessibility, focus states, and semantic meaning for screen readers. Using `div` for interactive elements breaks native keyboard navigation and accessibility tree mapping.
**Action:** Replace `div` based interactive controls with semantic `<button type="button">` elements. Apply CSS resets (`appearance: none; padding: 0; color: inherit;`) to override default button styling and maintain visual design while inheriting native button accessibility features. Ensure buttons have descriptive localized `aria-label` attributes, use `aria-hidden="true"` on inner decorative icons, and explicitly define `:focus-visible` states for clear keyboard navigation feedback.
