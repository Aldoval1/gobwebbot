## 2024-05-24 - Semantic Buttons for Custom Interactive UI Elements
**Learning:** In the application's Jinja templates, custom interactive elements (like map zoom controls in `licenses.html` and `official_businesses.html`) were implemented as `<div>` elements with `onclick` handlers, lacking proper keyboard accessibility, focus states, and screen reader announcements. Using a semantic `<button type="button">` inherently handles keyboard events (Space/Enter) and focusability, while specific CSS resets maintain the desired custom visual appearance without browser default styling getting in the way.
**Action:** When creating or refactoring custom interactive UI elements (e.g., zoom controls, custom toggles), always prefer semantic `<button type='button'>` tags over `<div>` elements. Ensure they feature descriptive `aria-label` attributes (e.g., localized to Spanish), `aria-hidden='true'` on inner decorative icons, explicit `:focus-visible` CSS rules for keyboard accessibility, and CSS resets (`appearance: none; padding: 0; color: inherit;`) to override default browser button styling.
